# Contributing to EAA Traffic Monitor

Thanks for your interest in contributing! This project is in early planning stages — see [README.md](README.md) for the project vision and [CLAUDE.md](CLAUDE.md) for current status and open decisions.

## Before you start

Since the tech stack and architecture are still undecided, it's worth opening an issue to discuss significant changes (new dependencies, architecture decisions, stack choices) before investing time in a pull request.

## How to contribute

1. Fork the repository and create your branch from `main`.
2. Make your changes.
3. Open a pull request against `main` using the pull request template.
4. Assign **@ericwait** as the reviewer/assignee on your pull request.
5. Address any review feedback. At least one approving review is required before a pull request can be merged.

## Reporting bugs or proposing features

Please use the issue templates provided when opening a new issue — they help capture the information needed to act on your report quickly.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold it.
